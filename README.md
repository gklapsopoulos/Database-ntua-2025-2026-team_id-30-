# ΝΟΣΟΚΟΜΕΙΟ ΥΓΕΙΟΠΟΛΙΣ

##### Εργασία Βάσεων Δεδομένων 6ου εξάμηνο ροης Λ του μαθήματος "Βάσεις Δεδομένων", ΗΜΜΥ ΕΜΠ

Καλωσορίσατε στην βάση δεδομένων του Νοσοκομείου Υγειόπολις. Στόχος της παρούσας ασκήσης είναι η προσομοίωση μιας ρεαλιστικής βάσης δεδομένων ενός νοσοκομείου και η αποδοτική διαχείρηση των δεδομένων που εισάγονται σε αυτήν. Η βάση διαχειρίζεται πληθώρα από κατηγορίες δεδομένων που χρειάζεται ενα νοσοκομείο, όπως στοιχεία των ασθενών του και διαφόρων ειδών προσωπικού. Ακόμη, περιέχει χρήσιμες πληροφορίες για την ταυτοποίηση νοσηλείων, διαγνώσεων, φαρμάκων, δραστικών ουσίων και πολλών άλλων, αποθηκεύοντας ταυτόχρονα και πληροφορίες για τις μεταξύ σχέσεις των διαφόρων κατηγοριών. Τα queries τις βάσεις επιταχύνονται με την χρήση ευρετηρίων.Σε αυτόν τον φάκελο υπάρχουν όλα τα απαραίτητα αρχεία που χρείαζονται για την εγκατάσταση τις βάσεις και της εισαγωγή δεδομένων σε αυτήν 



##### Παραδοχές 

1. Το κάθε τμήμα έχει έναν μοναδικό διευθυντή
2. Η κάθε κλίνη  του νοσοκομείου έχει μοναδικό id 
3. Κάθε κλίνη ανήκει σε μοναδικό τμήμα 
4. Ο τύπος του προσωπικού(staff\_type) μπορεί να πάρει μόνο τις τιμές:'doctor', 'nurse', 'administrative staff'
5. Οι ειδικότητες των γιατρών μπορούν να πάρουμ μόνο τις τιμές: 'Cardiology', 'Endocrinology', 'Gastroenterology', 'Neurology', 'Psychiatry', 'Neurosurgeon', 'Pediatrician', 'Gynecology', 'Anesthesiology', 'Oncology', 'Dermatology'
6. Οι βαθμίδες των νοσηλευτών μπορούν να πάρουν μόνο τις τιμές : 'nursing assistant', 'nurse', 'head nurse'
7. Οι ρόλοι του δοιηκητικού προσωπικού(role) μπορούν να πάρουν μόνο τις τιμές: 'Admissions Officer', 'Department Secretary', 'Appointment Scheduler', 'Accountant', 'Billing Specialist', 'HR Officer', 'Procurement Officer', 'Inventory Manager', 'Administrative Director', 'IT Support Officer'
8. Οι τύποι  κρεβατιών(type) μπορούν να παρούν μόνο τις τιμές : 'ICU', 'single-bed', 'multi-bed', 'Isolation', 'Recovery-bed', 'Double-room-bed'
9. Το status των κρεβατιών(status) μπορούν να πάρουν μόνο τις τιμές : 'occupied', 'not occupied', 'maintenance' 
10. Οι επεμβάσεις(surgery)  είναι υποκλάση των ιατρικών επεμβάσεων(hospitalization procedure)
11. Οι μόνοι γιατροί που επιτρέπεται να χειρουργούν είναι οι νευροχειρουργοί(Neurosurgeon) και γυναικολόγοι (Gynecology). 
12. Το κάθε rank αντιστοπροπεύεται από έναν αριθμό (1 έως 4) και ένα description. Μεγαλύτερος αριθμός συνεπάγεται καλύτερο rank.
13. Ένα γιατρός Α είναι επόπτης γιατρού Β αν και μόνο αν rank.int(A) > rank.int(B). Επομένως, η μέγιστη αλυσίδα εποπτείας έχει μήκος 4.  
14. Τα δεδομένα που υπήρχαν στα ελλήνικα δεν υπέστησαν αλλαγές ως προς το περιεχόμενο τους αλλά μεταφράστηκαν σε "Greeklish" με χρήση vba-script στο excel. Υπάρχει διαδέσιμο αρχείο "" στον φάκελο "" που περιέχει τον τρόπο με τον οποίο ένας ελληνικός χαρακτήρας μετατρέπεται σε αγγλικό
15. Στο query 6 για συγκεκριμένο ασθενή, βρίσκουμε τον μέσο όρο της κάθε αξιολόγησης του και όχι τον γενικό μέσο όρων όλων των αξιολογήσεων (όπως ζητάει στην εκφώνηση), γιατί με τον δεύτερο τρόπο η στήλη average\_of\_all\_hospitalizations έχει 1 στοιχείο και δεν μπορεί να είναι στο ίδιο πίνακα με τις υπόλοιπες στήλες που ζητάει το query, οι οποίες συνήθως έχουν παραπάνω από 1 στοιχεία.

#### ΦΟΡΤΩΣΗ ΔΕΔΟΜΕΝΩΝ

Για την ορθή φόρτωση των δεδομένων sql/install ώστε να φορτωθεί στην βάση το σχήμα, τα triggers και τα indexes. Εξαιτίας του γεγονότος ότι τα δεδομένα φορτώνονται στην βάση μέσω load θα πρέπει να πραγματοποιήσετε μια μικρή επεξεργασία του αρχείου των δεδομένων έτσι ώστε να βάλετε τα paths των αρχείων στον δικό σας. Ενδεικτικά βήματα παρουσιάζονται παρακάτω:
1. Δημιουργήστε έναν νέο φάκελο και κάντε export τα αρχεία από το zip εκεί
2. Ανοίξτε το αρχείο των δεδομένων sql/load με κάποιον editor(προτείνουμε το vscode)
3. Επιλέξτε και αντικαταστήστε τα paths των αρχείων μέχρι το όνομα του αρχείου του εκάστοτε csv με το path του φακέλου όπου τα αποθηκεύσατε(Προσοχή: Σε windows όπου δοκιμάστηκε η εντολή copy path αντέγραφε τη διεύθυνση χωρίζοντας τα αρχεία με \ το οποίο δημιουργούσε πρόβλημα. Αν συμβεί αντικαταστήστε με /
4. Αποθηκεύστε το αρχείο των δεδομένων
5. Τρέξτε όπως τα προηγούμενα με χρήση της εντολής source
